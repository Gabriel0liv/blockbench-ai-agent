# Blockbench AI Agent (MVP 1)

An experimental Blockbench plugin paired with a local backend server that allows controlling Blockbench model assets using declarative JSON commands generated by an agent.

## Project Structure

This project is a monorepo managed with **pnpm workspaces**:

- `packages/shared`: Shared TypeScript types, Zod validation schemas for AI commands, and structures.
- `apps/agent-server`: A Fastify local HTTP server that exposes a `/agent/run` endpoint to validate, parse, and mock AI-generated model commands.
- `apps/blockbench-plugin`: The Blockbench plugin interface, bundled to standard JavaScript via `tsup`, containing the Tools action loader and the isolated commands driver.

## Prerequisites

- [Node.js](https://nodejs.org/) (v18+)
- [pnpm](https://pnpm.io/) or `npx pnpm` (v10+)
- [Blockbench](https://www.blockbench.net/) (**Desktop app only**, required for communicating with the local server `http://127.0.0.1:3000`)

## Installation & Setup

1. Clone or download this project workspace.
2. Install all dependencies from the root directory:
   ```bash
   npx pnpm install
   ```

## Development

To compile and build all workspaces:
```bash
npx pnpm build
```

To run the local backend agent server in development mode:
```bash
npx pnpm dev:server
```
The server will run on `http://127.0.0.1:3000`.

To build the Blockbench plugin in watch mode for changes:
```bash
npx pnpm dev:plugin
```
This updates the compiled plugin at `apps/blockbench-plugin/dist/ai_model_agent.js` dynamically.

## Features and Execution Specs (MVP 1)

The following commands are fully implemented in the current MVP 1:
- `create_group`: Creates a bone/group. If no `parent` is specified, it defaults to a fallback group named `ai_generated`.
- `create_cube`: Creates a cube element. If no `parent` is specified, it defaults to `ai_generated`.
- `move_to_group`: Moves an existing element (cube or group) to another group.

### Strict Validation Rules
- **Parent Validation**: If `parent` is specified in any command but does not exist in the active Blockbench outliner, **execution is blocked** and an error is shown in the panel to protect model consistency.
- **Fallbacks**: Commands without a specified `parent` are automatically grouped inside a root-level bone called `ai_generated` instead of cluttering the project root.
- **Cube Size Constraints**: Cube volume size limits are validated (dimensions must be greater than zero, `from` coordinates must be strictly less than `to` coordinates, and dimensions on any axis cannot exceed 32 units).
- **Texturing/Painting**: The command `set_face_color` is defined in schemas and types, but **actual painting/texturizing is not implemented** in MVP 1 and is deferred to future milestones.

## Loading the Plugin in Blockbench

1. Open the **Blockbench Desktop App**.
2. Go to **File** > **Plugins...**
3. Click the three dots icon (`...`) at the top right, then select **Load Plugin from File**.
4. Browse to `<project-root>/apps/blockbench-plugin/dist/ai_model_agent.js` and select it.
5. A new option will appear under **Tools** > **AI Model Agent**. Click it to open the prompt dialog.
6. Select an active group in your outliner, type: *"cria dois chifres pequenos no grupo selecionado"*, and click **Run**.
7. Two unique cubes will be added in a new nested group. Press `Ctrl + Z` or click **Undo AI** to undo the operation.
